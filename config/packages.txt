# squashfs-tools and zstd are needed on the HOST by the ISO builder, not in
# the installed rootfs.  live-boot and live-config must be injected as an
# overlay by the dayshield-iso pipeline so they are present in the squashfs
# for live-boot operation but absent from the installed-system rootfs.
# Installing them here causes the initramfs to embed live-boot hooks that
# stall the installed system during boot (the hooks timeout looking for a
# live squashfs medium and drop to an emergency shell).
